# 비선형 구조

## 트리

### 정의
- 싸이클이 없는 그래프

### 용어
- 부모
- 자식
- 형제

- 깊이: 트리의 최대 레벨
- 차수: 어떤 노드에 연결된 자식 노드의 수
- 단말 노드: 제일 마지막에 위치한 노드(차수 = 0)
- 트리의 차수: 트리 노드 중 가장 큰 차수(가장 많은 자식 노드를 가진 노드의 차수(자식 노드의 수))

### 이진 트리
- 차수가 항상 2이하인 노드들로만 구성된 트리
- 최대 노드의 수 2^k-1 (k는 깊이)

### 이진 트리의 구조
- 정이진 트리: 모두 다 노드가 2개씩 채워진 트리
- 전이진 트리: 정이진 트리에서 한 쪽 방향 노드가 아예 존재하지 않는 트리
- 사향 이진 트리: Skewed

### 이진 트리의 운행법
- 전위(pre) 운행: root -> left -> right
- 중위(in) 운행: left -> root -> right
- 후위(post) 운행: left -> right -> root

### 수식의 표기법
- 전위 표기법: 연산자 -> 피연산자 -> 피연산자: +AB
- 중위 표기법: 피연산자 -> 연산자 -> 피연산자: A+B
- 후위 표기법: 피연산자 -> 피연산자 -> 연산자: AB+
> 괄호 쓰우고 빼기

## 해싱

### 정의
- 해싱 함수를 이용하여 레코드 키에 대한 해시 테이블 내에 홈주소를 계산하여 주어진 레코드에 접근하는 방식
- 직접 접근 파일을 구성할 때 사용됨
- 속도는 가장 빠르지만, **충돌 현상** 시 오버플로 해결의 부담이 가중되며, 많은 기억 공간을 요구

### 해싱 함수의 종류
- 제산
- 중간 제곱(mid-sqaure)
- 중첩(폴딩) - XOR
- 기수 변환 방법(Radix)
- 계수 분석 방법(Digit)

### 해싱 관련 용어
- 동의어
    - 동일한 홈주소로 인하여 충돌이 일어난 레코드들의 집합

- 슬롯
    - 한 개의 레코드들 저장할 수 있는 공감

- 충돌
    - 2개의 상이한 레코드가 똑같은 버킷으로 해싱되는 것을 의미
    - 충돌이 발생하면 항상 오버플로우가 발생
    - 버킷이 여러 개의 슬롯으로 구성될 때에는 충돌이 발생하여도 오버플로우가 발생하지 않을 수 있음

### 색인 순차 파일의 구성 - 정적 인덱싱
- 기본 영역
    - 데이터 레코드를 지정하는 부분

    - 기본 영역에 인덱스가 저장되는 부분

        - 트랙 인덱스
        - 실린더 인덱스

    - 한 블록 내에 레코드들이 모두 영역을 차지하여 추가적인 레코드 입력을 처리할 수 없을 때
    - 블록을 할당받아 이를 연결시키는 부분이며 실린더 오버플로 영역과 독립 오버플로 영역으로 구성됨

### VSAM (가상 기억 접근 방식) - 동적 인덱스
- 동적 인덱스 방법을 이용한 색인 순차 파일

# 관계 대수와 연산자

## 관계 대수와 관계 해석

### 관계 대수
- 원하는 정보와 그 정보를 어떻게 유도하는가를 기술하는 **절차적인** 방법
- 주어진 릴레이션 조작을 위한 연산의 집합
- 일반 집합 연산과 순수 관계 연산으로 구분됨
- 질의에 대한 해를 구하기 위해 수행해야할 연산의 순서를 명시

### 순수 관계 연산자 종류
- select(시그마 기호): 튜플 집합을 검색
- project(사람 다리 기호): 속성 집합을 검색
- join(나비 넥타이): 두 릴레이션의 공통 속성을 연결
- division(나누기): 두 릴레이션에서 특정 속성을 **제외한 속성만 검색**

### 선택
- 튜플을 선택하는 것이므로
- 연산자 기호는 시그마를 사용
- 수평적 연산
- 일본어 노 기호르 사용하기도 함

### 프로젝트
- 속성을 뽑아냄
- 파이를 사용

### 조인 
- 두 릴레이션을 연결하여 새로운 릴레이션을 만드는 연산

# SQL, DDL, DCL, View

## SQL, DDL

### SQL
- Structured Query Language
- DDL / DML / DCL

### DDL
- CREATE / ALTER / DROP
- ALTER에서 ADD, DROP을 사용함(열 추가, 삭제)

### DML
- SELECT / INSERT / UPDATE / DELETE

### DCL
- 데이터 제어 정의 및 기술에 사용되는 언어
- 불법적인 사용자로부터 데이터를 보호
- 무결성을 유지하고 데이터 복구 및 **병행 제어를 함**
- 종류
    - COMMIT
    - ROLLBACK
    - GRANT: 사용자에게 사용 권한을 부여
    - REVOKE: 사용 권한을 취소

### 뷰
- 사용자에게 접근이 허용된 자료만을 제한적으로 보여주기 위해 기본 테이블에서 유도되는 가상 테이블
- 물리적으로 존재 XX

### 뷰의 특징
- 뷰의 생성 시 CREATE문, 검색 시 SELECT 문을 사용
- 정의 변경시 **ALTER를 사용할 수 없고**, DROP을 사용
- 뷰를 이용한 또 다른 뷰 생성이 가능
- 하나의 뷰 제거 시 그 뷰를 기초로 정의된 다른 뷰도 함께 삭제됨
- 뷰에 대한 조작에서 삽입, 갱신, 삭제 연산은 제약이 따름 

### 뷰의 장점
- 논리적 데이터 독립성 제공
- 사용자 데이터 관리 편의성을 제공
- 접근 제어를 통한 보안을 제공

### 뷰의 단점
- ALTER VIEW 문으로 뷰의 정의 변경이 불가능함
- 삽입, 갱신, 삭제 연산에 제약이 따름

### 시스템 카탈로그
- 시스템 자신이 필요로 하는 여러 가지 객체에 관한 정보를 포함하고 있는 시스템 데이터베이스
- 데이터 사전, 메타 데이터라고도 함
- 사용자가 직접 갱신하는 것이 아님 (DBMS가 관리하는 것!!)

## 트랜잭션

### ACID
- 원자성
- 일관성
    - 시스템의 고정 요소는 트랜잭션 수행 전후가 같아야함
    - 트랜잭션 결과는 일관성을 유지
    - 500만원에서 10만원 이체 => 송수신자의 합계가 동일
- 격리성
    - 간섭 X
- 영속성
    - 영구히 데이터베이스에 트랜잭션 완료 결과 저장

### CRUD MATRIX
- 데이터베이스에 영향을 주는 생성, 읽기, 갱신, 삭제 연산의 프로세스와 테이블 간에 매트릭스를 만들어서 트랜잭션을 분석하는 도구
- 업무 프로세스와 데이터 간의 상관 관계 분석을 위한 것
- 행과 열의 교차점을 활용

### **즉각 갱신법**
- 데이터를 갱신하면 트랜잭션이 완료되기 전에 실제 데이터베이스에 반영하는 방법
- 회복 작업을 위해서 갱신 내용을 별도로 **LOG로 기록**
- Redo, Undo 모두 사용 가능

### 트랜잭션 연산
- COMMIT
- ROLLBACK
- **Recovery**

### 병행 제어
- 동시에 수행되는 트랜잭션들을 일관성 있게 처리하기 위해 제어하는 것
- 목적
    - 데이터베이스의 공유를 **최대화**
    - 데이터베이스의 일관성을 **최대화**
    - 시스템 활용도를 최대화
    - 사용자에 대한 응답 시간을 **최소화**

- 문제점
    - 갱신 분실, 비완료 의존성, 모순성, 연쇄 복귀

- 종류
    - 로킹(locking), 최적 병행 수행, 타임스탬프, 다중 버전 기법

### 타임 스탬프
- 트랜잭션이 DBMS로부터 유일한 타임스탬프를 부여 받음
- 동시성 제어를 위한 직렬화 기법으로 트랜잭션 간의 순서를 미리 정하는 방법

### 병행 제어 기법
- 로킹의 의미
    - 하나의 트랜잭션이 데이터를 액세스하는 동안 다른 트랜잭션이 그 데이터 항목을 액세스할 수 없도록 하는 병행 제어 기법

### 로킹의 특징
- 로킹의 단위가 커지면 로크의 수가 적어 관리가 쉬워지지만 병행성 수준이 낮아짐 / 공유도 감소
- 작으면 반대
- 로킹의 대상이 되는 객체(파일, 테이블, 필드, 레코드)의 크기를 로킹 단위

- 2단계 로킹
    - 직렬성은 보장하지만 교착 상태 예방은 불가능
    - **확장 단계와 축소 단계가 있음**

### 권한 부여 기법
- GRANT
- GRANT 권한 ON 데이터 객체 TO 사용자 WITH GRANT OPTION
- WITH GRANT OPTION: 사용자가 부여 받은 권한을 다른 사용자에게 다시 부여할 수 있는 권한을 부여
- 부여 가능한 권한(UPDATE, DELETE, INSERT, SELECT)

### REVOKE
- 권한 해제
- GRANT OPTION FOR: 다른 사용자에게 권한을 부여할 수 있는 권한을 취소
- CASCADE: 권한을 부여 받았던 사용자가 다른 사용자에게 부여한 권한도 연쇄 취소
- 부여 가능한 권한(UPDATE< DLEETE>, INSERT< SELECT>)

### 트래거
- 연쇄 반응
- 자동 처리되도록 하는 석

### 분산 데이터베이스
- 위치 투명성
    - 사용자들이 물리적 위치를 알 필요가 없음
- 중복 투명성
    - 사용자에게 말하지 않고, 자료를 복제해서 여러 곳에 저장
- 병행 투명성
    - 다중 사용자들이 같은 자료를 공유할 수 있음
- 장애 투명성(Failure)
    - 사용자들은 어느 위치의 시스템에 장애가 발생했는지 알 필요가 없음

### 장단점
- 질의 처리 시간의 단축
- 데이터 공유성 신뢰성 가용성 향상
- 점진적 시스템 용량 확장 요이
- 지역 자치성 향상

- 개발 비용 증가
- 요류 발생 증가
- 전체적인 시스템 성능 저하(통신망 저하)
- 하드웨어 구매 비용 증가

### 지역 자치성
- 데이터베이스 중앙 관리자 외에 각 지역에 담당 관리자를 두는 것

### 병렬 처리
- 하나 이상의 처리를 분산되어잇는 데이터베이스에 분산 처리하여 처리 속도를 높임
