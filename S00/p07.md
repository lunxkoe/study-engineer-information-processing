## UML(Unified Modeling Language)

### UML
- 시스템 분석, 설게, 구현 등의 산출물을 명세화, 시각화, 문서화 할 때 사요하는 모델링 기술과 방법로을 통합하여 만든 **범용 모델링 언어**
- Rumbaugh의 OMT 방법론과 Booch 방법론, Jacobson의 OOSE 방법론을 통합

### **럼바우 객체지향 분석 기법**
- 객체 모델링
    - 정보 모델링
    - 시스템에서 요구되는 객체를 찾아내어 속성과 연산 식별 및 객체들 간의 관계를 규정
    - **객체를 다이어그램으로 표시**
    
- 동적 모델링
    - 제어 흐름
    - 상호작용
    - 동작 순서
    - 시간 흐름에 따라 **상태 다이어그램**으로 표시

- 기능 모델링
    - 프로세스 간의 자료 흐름을 표시
    - 어떤 데이터를 입력하여 어떤 결과를 가져올 수 있을지 표현
    - **자료 흐름도(Data Flow Diagram)**

### UML의 특성
- 비주얼화(시각화)
- 문서화
- 명세화: 구체적으로 표현
- 구축

### UML 소프트웨어에 대한 관점
- **기능적** 관점
    - 사용자 측면에서 본
    - 사용 사례 모델링 
    **Use Case Diagram**

- **정적** 관점
    - 내부 구성 요소 사이의 **구조적 관계**
    - 객체, 속성, 연관관계, 오퍼레이션의 시스템 구조를 나타냄
    - **Class Diagram**

- **동적** 관점
    - 시스템 내부 동작
    - **Sequence Diagram**
    - **State Diagram**
    - ***Activity Diagram**

### UML의 기본 구성
- 사물
    - 객체 간의 관계 형성 대상

- 관계
    - 객체 간의 연관성

- 다이어그램
    - 정적 모델: 구조 다이어그램
    - 동적 모델: 행위 다이어그램

### 스테레오 타입
- UML에서 제공하는 기본 요소 외에 추가적인 확장 요소를 표현할 때 사용함
- <<>>(길러멧) 안에 확장 요소를 적임

### UML 접근 제어자
- public: +
- private: -
- protected: #
- package: ~

### 연관 관계 다중성 표현
- 1: 1 개체 연결
- * 또는 0..*: 0이거나 그 이상 객체 연결
- 1..*: 1이거나 그 이상 객체 연결
- 0..1: 0이거나 1 객체 연결
- 1, 3, 6: 1이거나 3이거나 6 객체 연결
- n: n개 객체 연결
- n..*: n이거나 n이상 객체 연결

## UML 다이어그램

### 구조적 다이어그램 - 멈춰 있는 느낌이 매우 강함
- 정적이고, 구조 표현을 위한 다이어그램

- 유형
    - 클래스 다이어그램(정적 - 클래스 사이의 관계)
    - 객체 다이어그램(객체 정보)
    - 복합체 구조 다이어그램(클래스와 컴포넌트 내부)
    - 배치 다이어그램(Deployment - 소프트웨어, 하드웨어, 네트워크 물리 구조 표현)
    - 컴포넌트 다이어그램

### 행위 다이어그램 - 시간
- 동적이고, 순차적인 표현을 위한 다이어그램

- 유스케이스 다이어그램
    - **사용자 관점**에서 시스템 행위를 표현

- 활동 다이어그램
    - 업무 처리 과정이나 연산이 수행되는 과정을 표현

- 상태 머신 다이어그램
    - 객체의 생명주기를 표현함
    - 동적 행위 모델링이지만 특정 객체만을 다룸

- 콜라보레이션 다이어그램: 시퀀스 다이어그램과 비슷
- 통신 다이어그램
- 타이밍
- 즉, 시간과 관련이 있음

## 클래스 다이어그램 관계 표현

### 클래스 다이어그램
- 구성
    - 클래스 이름
    - 속성
    - 오퍼레이션: 리턴 타입
    - 속성과 오퍼레이션은 접근 제어자

### UML 관계 표현
- 단방향 연관 관계
    - 실선 한 방향

- 양방향 연관 관계
    - 실선(화살표가 없음)

- 의존 관계
    - 연관관계와 같지만 메소드를 사용할 때와 같이 매우 짧은 시간만 유지
    - 점선 한 방향

- 일반화 관계 (상속)
    - 실선(화살표가 비어있는 삼각형)
    - **IS-A**
    - 상속 관계에서 사용

- 집합/포함 관계
    - 클래스 사이 전체나 부분이 같은 관계
        - 실선에 마름모 비어있는 화살표

    - 전체/부분 객체 라이프타임 의존적(전체 객체 삭제 -> 부분 객체 삭제)
        - 실선에 마름모 채워진 화살표

- 실체화 관계
    - 점선 화살표가 비어있는 삼각형

### UML 연관 관계
- 한 사물의 객체가 다른 사물의 객체와 연결된 것을 표현함
- 이름: 관계의 의미를 표현하기 위해 이름을 가질 수 있음
- 역할명: 수행하는 역할을 명시적으로 이름을 가질 수 있음
- 다중성 표시: 1..* 같은
- 연관 관계명: 상담

### UML 의존 관계
- 연관관계와 같지만 메소드를 사용할 때와 같이 매우 짧은 시간만 유지됨
- 영향을 주는 객체에서 영향을 받는 객체 방향으로 점선 화살표를 연결
    - 자동차 - - -> 연료

### UML 집합 관계
- 전체 클래스 방향에 빈 마름모로 표시
    - or 관계에 놓이면 선 사이를 점선으로 잇고 {or}를 표시
    - 학교 (비어있는 마름모)--- (학생, 교사)

### UML 포함 관계
- 부분 객체가 전체 객체에 속하는 강한 집합 연관의 관계를 표현
- 채워져있는 마름모
- 책상 (채워진 마름모)---- (책상, 상판)

### UML 실체화 관계 - 인터페이스 - 구현체 관계
- 인터페이스와 실제 구현된 일반 클래스 간의 관계
- 인터페이스 (빈 삼각형)- - - (구현체들)

## Use Case Diagram

### 개념
- 구성
    - Actor
    - Use Case

### 요소
- **시스템 경계**
    - Use Case들의 범위가 됨
    - 큰 구모의 객체로 구현되는 존재

- 액터
    - 서비스를 사용하는 외부 객체
    - 특정한 사례(Use Case)를 실행하도록 요구할 수 있는 존재

- 유스 케이스
    - 시스템이 제공해야하는 개별적인 서비스 기능
    - 서비스는 특정 클래스의 멤버 함수로 모델링

- 접속 관계
    - 액터/유스케이스 또는 유스케이스/유스케이스 사이에 연결되는 관계
    - 액터나 유스케이스가 다른 유스케이스의 서비스를 이용하는 상황을 표현함

- 사용 관계
    - 여러 개의 유스케이스에서 공통으로 수행해야 하는 기능을 모델링하기 위해 사용

- 확장 관계

### Use Case Diagram 작성 단계
- 액터 식별
- Use Case 식별
- 관계
- Use Case 구조화
