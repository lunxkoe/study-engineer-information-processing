# UI 표준 및 지침

## UI 표준 지침

### UI 분야
- 표현
- 정보 제공과 전달 분야
- 기능 분야

### UI 특징
- 실사용자의 만족도에 직접 영향을 줌
- 실사용자가 수행해야할 기능을 구체적으로 제시
- UI 설계 전 소프트웨어 아키텍처를 우선 숙지

### 기능
- 사용자 입력과 검증
- 에러 처리와 에러 메시지 처리
- 도움과 프롬프트 제공(커서 같은 거)

## UI 설계

### UI 설계 원칙
- 직관성: 
    - 누구나 쉽게 이해 사용

- 유효성:
    - 사용자의 목적을 정확히 달성할 수 있도록 유용하고 효과적
    
- 학습성:
    - 사용자가 쉽게 배우고 익힐 수 있어야 함
    - 로그인 창 위치 (오른쪽 위)

- 유연성:
    - 사용자 요구를 최대한 수용하면서 오류 줄이기


### UI 설계 지침
- 사용자 중심
- 일관성
- 단순성
- 가시성
- 표준화
- 접근성:
    - 사용자의 직무, 성별 나이등 다양한 계층 수용

- 결과 예측 가능
- 명확성
- 오류 발생 해결:
    - 오류가 발생하면 사용자가 상황을 정확히 인지할 수 있어야함

## UI 설계 - 시스템 아키텍처를 분석하고 UI를 설계함

### UI 설계 단계
- 문제 정의
- 사용자 모델 정의
- 작업 분석
- 컴퓨터 오브젝트 및 기능 정의
- 사용자 인터페이스 정의
- 디자인 평가
    - GOMS: 인간이 어떤 행위를 할지 예측하여 문제를 해결
    - Heuristics: 논리가 아닌 어림짐작으로 문제를 해결

### UI 상세 설계 단계
1. UI 메뉴 구조 설계
2. 내/외부 화면과 폼 설계
3. UI 검토 수행

### 상세 설계 - 시나리오 작성 원칙
- UI 전체적 기능, 작동 방식을 개발자가 쉽게 이해할 수 있도록 구체적으로 작성
- Tree 구조나 Flowchart 표기법을 이용
- 공통 적용이 가능한 UI 요소와 상호 작용을 일반적인 규칙으로 정의
- 흐름 및 순서, 분기, 조건, 루프를 명시
- 예외 상황과 관한 사례를 정의
- 기능별 상세 기능 시나리오를 정의
- 작성 요건
    - 완전성
    - 일관성
    - 이해성
    - 가독성
    - 수정 용이성
    - **추적 용이성**

    - ***효과성 아님**
